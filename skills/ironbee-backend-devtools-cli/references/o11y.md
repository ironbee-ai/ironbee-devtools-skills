# O11y Tools

Session-pinned W3C trace context. Pin a trace id once and every subsequent `request_*` call auto-injects it as `traceparent` — no per-call plumbing required.

**Trace id precedence (highest to lowest):**
1. Per-call `--trace-id` argument on a `request_*` tool
2. `_metadata.traceId` from the MCP client (the orchestrator's correlation root)
3. Session pin (set via `o11y new-trace-id` or `o11y set-trace-context`)
4. Random per-call

The MCP client's metadata trace id outranks the session pin so the orchestrator's correlation boundary is never accidentally broken by the agent.

---

## new-trace-id

Generate a fresh 32-hex W3C trace id and pin it on the session. Subsequent `request_*` calls auto-inject it as `traceparent`. Useful at the start of a correlated flow.

```bash
ironbee-backend-devtools-cli o11y new-trace-id
```

No arguments. Returns `{ traceId }`.

---

## set-trace-context

Set or clear the session-pinned trace id and/or tracestate. At least one of the two fields is required. Passing an empty string clears that field.

```bash
ironbee-backend-devtools-cli o11y set-trace-context [--trace-id <hex>] [--trace-state <string>]
```

| Argument | Type | Required | Default | Description |
|----------|------|----------|---------|-------------|
| `--trace-id` | string | Conditional | - | 32-hex W3C trace id. Empty string clears the pin. |
| `--trace-state` | string | Conditional | - | W3C `tracestate` header value. Empty string clears it. |

At least one of `--trace-id` or `--trace-state` must be supplied.

---

## get-trace-context

Inspect the session-pinned trace context. Does NOT reflect per-call `--trace-id` arguments or `_metadata.traceId` from the MCP client — only the session pin.

```bash
ironbee-backend-devtools-cli o11y get-trace-context
```

No arguments. Returns `{ traceId, traceState }`.
